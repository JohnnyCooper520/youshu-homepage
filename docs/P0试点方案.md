# 有数 P0 试点方案

> 版本：2026-06-25
> 定位：东方命理 AI 人生顾问
> 试点目标：用最小成本验证「内容获客 -> 网页起盘 -> 付费解读 -> 分享/复购」是否成立。

## 1. P0 核心判断

有数现阶段不先做完整 App，也不先重投入供应链和真人师傅。P0 只验证三件事：

1. 用户是否愿意因为一段命盘洞察留下来。
2. 用户是否愿意为更深入的命盘、问事、今年运势付费。
3. 结果内容是否有截图、分享、复盘的传播潜力。

P0 的产品形态先以网页为主，避免过早进入 App 上架、算法备案、应用备案和复杂支付体系。

## 2. 主体策略

短期使用境内公司验证，不急于一开始注册香港公司。

境内公司适合承担早期试点：

- 方便做 ICP 备案、合同、基础收款和国内团队协作。
- 适合先跑中文网页 MVP、内容账号、用户反馈和首批订单。
- 能快速验证产品和转化，不把时间消耗在公司架构上。

但从第一天开始，要按未来可拆分海外业务来设计：

- 品牌、域名、用户协议、隐私政策不要写死为只服务中国大陆用户。
- 订单、用户、内容、支付记录要能区分国内与海外用户。
- 产品表达保持「命理文化 + AI 解读 + 决策参考」，避免宗教仪式、开光、化解、烧香、还愿等高风险表达。

当海外出现稳定付费后，再注册香港公司承接海外业务：

- 香港公司作为海外用户签约主体、收款主体和海外应用商店主体。
- 境内公司继续承担研发、内容生产、顾问规则沉淀和国内业务。
- 未来形成「境内公司做中国业务，香港公司做海外业务」的双主体结构。

结论：境内公司先跑，香港公司后置；但业务架构从 P0 就预留海外拆分口。

## 3. P0 产品范围

P0 只做四个购买选择：

| 产品 | 人民币价格 | 美元参考价 | 作用 |
| --- | ---: | ---: | --- |
| 命盘报告 | 29.9 元 | 约 4.2 美元 | 入门型产品，帮助用户认识自己的性格底色、用力方式和关系惯性。 |
| 问事解惑 | 29.9 元 | 约 4.2 美元 | 事件型产品，围绕一件具体问题给出趋势、风险边界和下一步建议。 |
| 今年运势解读 | 199 元 | 约 28 美元 | 核心收费产品，从起盘日起向后看完整 12 个月。 |
| 年度会员 | 299 元/年 | 约 42 美元/年 | 长期账户产品，包含命盘、今年运势、问事额度、月度趋势、追问和历史归档。 |

网页和报告从 P0 起支持三种语言：

- 简体中文：默认中文市场版本。
- 繁体中文：为港澳台和海外华语用户保留入口。
- English：为海外试点和后续香港主体承接业务预留。

价格展示采用「人民币 + 美元参考价」双口径，正式支付价格以后以用户所在地区、支付通道和汇率策略为准。

P0 不做：

- 线上烧香拜佛。
- 开光、法事、还愿、化解灾祸。
- 真人大师视频咨询。
- 小程序版本。
- 完整 App。

## 4. 产品表达原则

前端营销可以有情绪表达，但支付、报告、协议和免责声明必须克制。

允许表达：

- 命盘有数，选择有光。
- 先看格局，再看选择。
- 仅供参考，辅助决策。
- 看趋势、看边界、看下一步。

避免表达：

- 包准、必应、保证复合、保证发财。
- 不化解会出事。
- 改命、转运、开光、消灾。
- 以恐惧、焦虑、威胁推动付费。

高风险主题，如医疗、法律、投资、情绪危机，只做一般性提醒和专业机构引导，不给确定性判断。

## 5. P0 验证路径

### 第 1 周：网页 MVP 和内容种子

- 完成 C 端网页第一版。
- 明确四个购买产品和会员权益。
- 做 10 条小红书/短视频图文选题。
- 设计 3 套可截图的命盘洞察样式。

### 第 2 周：付费闭环

- 接入最轻支付方案。
- 完成订单记录和报告归档的最小后台方式。
- 跑第一批真实用户，优先人工/半自动交付，避免一开始做复杂系统。

### 第 3 周：内容获客测试

- 每天发布内容。
- 测试不同钩子：事业、感情、财务、内耗、今年运势。
- 观察用户是否愿意进入网页、起盘、询价、付费。

### 第 4 周：复盘和是否继续

- 统计访问、起盘、支付、退款、复购、分享。
- 复盘哪类问题最容易付费。
- 判断下一步是强化 AI 报告、补后台，还是继续打磨内容账号。

## 6. P0 成功指标

最低通过线：

- 有真实陌生用户完成起盘。
- 有真实陌生用户付费。
- 至少一个产品被反复购买或咨询。
- 用户反馈报告「像我」「有用」「愿意继续看」。

较好信号：

- 命盘报告能稳定卖出。
- 今年运势 199 元有人购买。
- 用户愿意截图分享。
- 用户出现复问、追问和报告归档需求。

停止信号：

- 内容账号无法带来陌生用户。
- 用户只看免费洞察，不愿付费。
- 报告反馈泛、虚、像模板。
- 付费后投诉集中在「不准」「没用」「像 AI 套话」。

## 7. 接下来优先做什么

接下来不急着继续加页面，先把 P0 付费验证链路补齐。

优先级如下：

1. 定义第一版命盘报告结构。
2. 定义问事解惑的输入表单和输出结构。
3. 定义今年运势 199 元报告的目录和样例。
4. 确定支付后的交付方式：自动生成、人工审核、还是半自动。
5. 做「我的报告」最小账户逻辑：用户能找回已购买报告。
6. 写隐私政策、用户协议、免责声明、退款规则。
7. 做第一批内容选题和落地页转化路径。

P0 的关键不是把功能做全，而是让用户愿意为第一份真实报告付钱。
